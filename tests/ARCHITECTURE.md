## Test Flow Architecture

```
┌─────────────────────────────────────────────────────────────────┐
│                        Test Execution Flow                       │
└─────────────────────────────────────────────────────────────────┘

1. 🚀 Developer runs: npm test
                      │
                      ▼
2. 📝 Vitest loads: vitest.config.ts
                      │
                      ▼
3. 🔧 Runs setup: tests/setup.ts
                      │
                      ├─► Loads .env.test
                      ├─► Sets NODE_ENV=test
                      └─► Configures globals
                      │
                      ▼
4. 🔍 Discovers tests: **/__tests__/**/*.test.ts
                      │
                      ▼
5. 🏃 Executes tests (parallel, max 5 concurrent)
                      │
                      ├─► ProjectRepository.test.ts (20 tests)
                      ├─► AreaRepository.test.ts (15 tests)
                      ├─► ProjectTeamRepository.test.ts (15 tests)
                      └─► smoke.test.ts (4 tests)
                      │
                      ▼
6. 📊 Generates reports
                      │
                      ├─► Console output (verbose)
                      ├─► Coverage report (coverage/)
                      └─► UI dashboard (--ui flag)
                      │
                      ▼
7. ✅ Enforces coverage thresholds
                      │
                      ├─► Lines: 80%
                      ├─► Functions: 80%
                      ├─► Branches: 75%
                      └─► Statements: 80%
                      │
                      ▼
8. 🎉 Reports results: Pass/Fail
```

## Test Isolation Pattern

```
┌─────────────────────────────────────────────────────────────────┐
│                    Test Isolation Pattern                        │
└─────────────────────────────────────────────────────────────────┘

describe('ProjectRepository', () => {
  
  ┌─────────────────────────────────┐
  │      Test Suite Setup           │
  ├─────────────────────────────────┤
  │ let db: SupabaseClient;         │
  │ let repo: ProjectRepository;    │
  │ let projectIds: string[] = [];  │
  └─────────────────────────────────┘
                │
                ├─► beforeEach() ──► Fresh setup for each test
                │                    • New client
                │                    • New repository
                │
                ├─► it('test 1') ───► Isolated test
                │   ├─► Create data
                │   ├─► Test logic
                │   └─► Track IDs
                │
                ├─► it('test 2') ───► Independent test
                │   ├─► Create data
                │   ├─► Test logic
                │   └─► Track IDs
                │
                └─► afterEach() ────► Cleanup everything
                    └─► Delete all created data
                        (respects foreign keys)
});
```
